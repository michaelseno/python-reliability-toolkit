# AWS Phase D: Lambda Dashboard Builder

This phase adds an event-driven index refresh in AWS while CI publishes the latest dashboard HTML.

## Created Resources

- Lambda function: `rk-dashboard-builder`
- Lambda IAM role: `rk-lambda-dashboard-role`
- SNS topic: `arn:aws:sns:us-east-1:463470948609:rk-reliability-events`
- S3 event trigger on bucket `rk-reliability`
  - event: `s3:ObjectCreated:*`
  - prefix: `runs/`
  - suffix: `run.json`

## Lambda Behavior

On each `run.json` upload, Lambda:

1. Lists all `runs/*/run.json` objects
2. Builds summary metrics and latest runs index
3. Writes:
   - `index/latest_runs.json`
   - `index/window_analytics_30d.json`
   - `index/latest_build.json`

Dashboard HTML is generated by CI from repo templates and uploaded directly to S3.

## Notifications

Lambda publishes refresh success/failure notifications to SNS topic `rk-reliability-events`.

To subscribe email:

```bash
aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:463470948609:rk-reliability-events \
  --protocol email \
  --notification-endpoint you@example.com \
  --profile rk-reliability
```

Then confirm subscription from the email sent by AWS SNS.

## Environment Variables

- `RK_BUCKET=rk-reliability`
- `RK_RUNS_PREFIX=runs/`

## Public URL

- `https://rk-reliability.s3.amazonaws.com/dashboard.html`

## CI Integration

Scheduled workflow uploads run artifacts, index files, and rendered dashboard pages to S3.
Lambda refreshes derived index JSON only and does not overwrite dashboard HTML.
